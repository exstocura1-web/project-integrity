# Security Policy

## Supported Versions

Security updates are applied to the `main` branch.

## Reporting a Vulnerability

If you discover a security issue, do not open a public GitHub issue.

- Email: `michael@exstocura.com`
- Subject line: `Security vulnerability report - project-integrity`
- Include:
  - Affected component and file path(s)
  - Reproduction steps
  - Expected vs actual behavior
  - Impact assessment
  - Any suggested remediation

## Response Targets

- Initial acknowledgement: within 1 business day
- Triage and severity assessment: within 3 business days
- Remediation target:
  - Critical: 24-72 hours
  - High: within 7 days
  - Medium/Low: next planned release cycle
